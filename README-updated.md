# ねこレコ

## 概要

**ねこレコ** は、たくさんの猫と暮らす家庭や保護猫活動者向けの、猫ごとの個体管理アプリです。

ワクチン、駆虫薬、通院、体重、フード、保険請求、記念日、家族への申し送りなどを、猫ごとにわかりやすく記録・確認できるようにします。

単なる記録アプリではなく、**「猫ごとの情報を記録し、次にやることを忘れない」** ことを目的とします。

---

## コンセプト

> 猫ちゃんごとの体調・通院・ごはん・記念日を、家族みんなでわかりやすく管理できるアプリ。

多頭飼いでは、猫の数が増えるほど情報管理が難しくなります。

- ワクチンや駆虫薬の次回予定を忘れてしまう
- 偏食の子が食べてくれたごはんの銘柄を忘れてしまう
- 家族にお世話を任せるとき、注意事項を伝えきれない
- 通院後に「次回は何週間後」と言われたのに忘れてしまう
- ペット保険の請求が済んでいるかわからなくなる
- 診察室で測った体重をその場で残したい
- 誕生日やうちの子記念日を家族でお祝いしたい

ねこレコは、こうした日常の困りごとを減らすためのアプリです。

---

## MVP v1 の目的

ねこレコ MVP v1 では、多頭飼い家庭が猫ごとの基本情報・記録・次回予定を管理できる最小構成を実装します。

MVPでは、家族共有や留守番モード、PDF出力、体重グラフなどは将来拡張とし、まずは以下の体験を実現します。

- 猫を登録できる
- 複数匹の猫を管理できる
- 猫ごとに詳細情報を確認できる
- 体重、通院、フード、保険、メモを記録できる
- ワクチン、駆虫薬、通院、記念日の予定を忘れない
- ホームで今日やることがわかる

## 主な画面

### 1. 初回オンボーディング

初回起動時に、最初の猫を登録するための導線です。

登録項目：

- 写真
- 名前
- 生年月日
- 性別
- うちの子記念日
- 猫種・血統
- 毛色・柄

登録完了後は、以下を選択できます。

- 詳しい情報も入力する
- あとで入力する

### 2. ホーム画面

ホーム画面は、今日やることを確認するための画面です。

表示内容：

- 今日のねこたち
- 今日やること
- 近日の予定
- ねこ一覧ミニ表示
- 情報追加の案内カード
- ＋ 記録する

### 3. 猫一覧画面

登録済みの猫を2列グリッドで一覧表示します。

表示内容：

- 写真
- 名前
- 年齢
- 性別
- 毛色・柄
- 次の予定
- 注意アイコン
- 未完了タスク数

### 4. 猫詳細画面

1匹の猫について、プロフィール・医療・ごはん・履歴・保険・メモをまとめて確認できます。

タブ構成：

1. 概要
2. 医療
3. ごはん
4. 履歴
5. 保険
6. メモ

猫詳細画面からは、対象猫を固定した状態で **「＋ この子の記録を追加」** ができます。

### 5. 記録追加

記録追加の入口は3つです。

- ホーム画面：＋ 記録する
- 猫詳細画面：＋ この子の記録を追加
- 記録タブ：記録タイプ選択

MVPで扱う記録タイプ：

- 体重
- 通院
- フード
- 保険
- メモ
- 投薬
- 体調

猫詳細から記録する場合は、対象猫が決まっているため猫選択をスキップします。

### 6. 通知・リマインダー

MVPで扱う通知：

- ワクチン
- 駆虫薬
- 通院予定
- 投薬
- 誕生日
- うちの子記念日
- 保険請求

デフォルト通知：

| 通知対象 | タイミング |
|---|---|
| ワクチン | 7日前・前日・当日 |
| 駆虫薬 | 3日前・当日 |
| 通院予定 | 前日・当日 |
| 投薬 | 指定時刻 |
| 誕生日 | 7日前・当日 |
| うちの子記念日 | 7日前・当日 |
| 保険請求 | 未請求のまま3日後 |

## データモデル方針

### Cat は軽く保つ

`Cat` は基本プロフィールのみを持ちます。

医療、フード、保険、お世話情報はプロフィール系モデルに分けます。

- `Cat`
- `CatMedicalProfile`
- `CatFoodProfile`
- `CatInsuranceProfile`
- `CatCareProfile`

### 記録は CatRecord に集約する

日付つきの出来事はすべて `CatRecord` 系として扱います。

- `WeightRecord`
- `HospitalVisitRecord`
- `VaccineRecord`
- `DewormingRecord`
- `MedicationRecord`
- `FoodRecord`
- `HealthConditionRecord`
- `InsuranceClaimRecord`
- `MemoRecord`
- `CareRecord`

### 予定・タスク・通知は分ける

- 予定：未来の日付として登録されているもの
- タスク：ホーム画面で「今日やること」として表示するもの
- 通知：端末に実際に通知するもの
